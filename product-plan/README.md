## Как использовать

### Вариант A: Посекционная реализация (рекомендуется)

Реализуйте проект этап за этапом. Каждый этап самодостаточен и может быть протестирован отдельно.

1. Скопируйте промпт из `prompts/section-prompt.md`
2. Подставьте переменные `SECTION_NAME`, `SECTION_ID`, `NN` для нужной секции
3. Передайте промпт coding-агенту вместе с файлами из соответствующей папки
4. Начните с этапа 01 (Shell) и продвигайтесь по порядку

**Порядок реализации:**

| Этап | ID секции | Описание |
|------|-----------|----------|
| 01 | shell | Хедер, футер, навигация |
| 02 | glavnaya-stranitsa | Главная страница (лендинг) |
| 03 | katalog-treningov | Каталог тренингов |
| 04 | stranitsa-treninga | Страница отдельного тренинга |
| 05 | korporativnye-programmy | Корпоративные программы и консалтинг |
| 06 | klienty-i-keysy | Клиенты и кейсы |
| 07 | komanda-o-nas-kontakty | Команда, О нас и Контакты |

### Вариант B: Полная реализация (one-shot)

Реализуйте весь сайт за одну сессию.

1. Скопируйте промпт из `prompts/one-shot-prompt.md`
2. Передайте его coding-агенту вместе со всем содержимым `product-plan/`
3. Агент реализует все этапы последовательно

## Советы

- **Начинайте с Shell.** Хедер и футер — обёртка для всех остальных секций. Без них невозможно проверить навигацию.
- **Используйте дизайн-токены.** CSS custom properties из `tokens.css` обеспечивают единообразие. Подключите их глобально.
- **Компоненты props-based.** Все компоненты получают данные через props. Не импортируйте данные напрямую в компоненты — это упрощает интеграцию с любым бэкендом.
- **TypeScript-интерфейсы — контракт.** Файлы из `data-shapes/` определяют, какие данные ожидает UI. Бэкенд должен предоставить данные в этом формате.
- **Tailwind CSS v4.** Проект использует Tailwind CSS v4. Не создавайте `tailwind.config.js` — конфигурация происходит через CSS.
